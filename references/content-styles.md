# 内容风格指南

## 风格一：产品软文

以故事、场景、情感切入，自然植入产品信息。读者读完感觉是一篇有价值的内容，而非广告。

### 结构模板

```
1. 场景导入（100-200字）
   - 描绘一个读者能共鸣的生活场景或痛点
   - 不提产品，只讲故事

2. 话题展开（300-400字）
   - 围绕场景深入探讨
   - 引入行业趋势或社会现象（结合搜集的热点素材）
   - 建立"需求感"

3. 产品自然出场（400-500字）
   - 以"解决方案"或"发现"的方式引入产品
   - 突出2-3个核心卖点
   - 用具体细节代替空洞形容（如"经过72小时慢炖"而非"好吃"）

4. 体验描写（300-400字）
   - 感官细节：视觉、嗅觉、触觉、味觉
   - 真实使用场景还原
   - 适当引用用户评价（来自搜集素材）

5. 价值升华（200-300字）
   - 回归生活方式或价值观
   - 品牌理念自然呈现
   - 引导关注/购买，语气柔和不硬推
```

### 标题套路

- 场景型：`"下班回家的第一口热汤，治愈了我整天的疲惫"`
- 发现型：`"被朋友安利了这款XX，用了一个月说说真实感受"`
- 反常型：`"为什么越来越多人放弃大牌，选择这个小众品牌？"`
- 清单型：`"宅家必备的5样好物，第3个真的离不开了"`

### 写作禁忌

- 不开头就提产品
- 不使用"最好""第一""绝对"等绝对化用语
- 不贬低竞品
- 不虚构用户评价
- 不使用过多感叹号

## 风格二：种草测评

模拟真实用户的体验视角，以测评口吻推荐产品。强调真实感和专业感。

### 结构模板

```
1. 开箱/初印象（200-300字）
   - 购买动机（为什么买/怎么发现的）
   - 包装与外观第一印象
   - 设定期望值

2. 使用体验（500-600字）
   - 详细使用过程
   - 按维度逐项评价（如食品：味道、口感、配料表、分量）
   - 多场景测试
   - 配合 IMG_PLACEHOLDER 展示产品细节

3. 优缺点总结（200-300字）
   - 优点3-5个，具体化
   - 缺点1-2个，保持客观（增加可信度）
   - 用表格或列表呈现

4. 横向对比（200-300字）
   - 与同类产品简单对比（不贬低，只说差异）
   - 适合什么人群
   - 不适合什么人群

5. 购买建议（100-200字）
   - 性价比分析
   - 推荐购买渠道/时机
   - 总结一句话推荐语
```

### 标题套路

- 测评型：`"XX产品深度测评：用了30天，优缺点全说清"`
- 对比型：`"XX vs XX，同价位到底选谁？亲测告诉你"`
- 种草型：`"闺蜜推荐的XX，用完我直接囤了3份"`
- 避坑型：`"买XX前一定要看！这几个坑我替你踩了"`

### 写作要点

- 第一人称叙述
- 多用具体数据（"用了30天""第3次回购"）
- 优缺点并存增加可信度
- 适度使用口语化表达
- 评分制（如：味道 9/10、颜值 8/10）
